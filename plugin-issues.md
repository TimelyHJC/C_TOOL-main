## 图例说明

### 严重程度 (Severity)

| 标识 | 含义 |
|------|------|
| 🔴 严重 | 导致崩溃、数据丢失或核心功能不可用 |
| 🟠 高 | 重要功能异常，有明确 workaround |
| 🟡 中 | 一般功能缺陷或性能问题 |
| 🟢 低 | UI 细节、日志、文档等轻微问题 |

### 状态 (Status)

| 标识 | 含义 |
|------|------|
| 📝 待确认 | 已报告，待复现或确认 |
| 🔧 修复中 | 正在修复 |
| ✅ 已修复 | 已修复，待验证 |
| 🧪 验证中 | 修复后正在验证 |
| 📦 已发布 | 修复已合入发布版本 |
| ❌ 无法复现 | 无法复现，暂时关闭 |
| 🚫 不修复 | 确认为预期行为或暂不修复 |

## 使用说明

1. **新增问题**：在对应插件表格中新增一行，按模板填写。
2. **更新状态**：修改 `状态` 列，并在 `备注` 中补充进展。
3. **问题关闭**：状态置为 `✅ 已修复` 或 `🚫 不修复`，并在备注中说明原因。
4. **编号规则**：使用 `P{插件缩写}-{序号}` 格式，如 `PAAA-001`、`PPLUGIN-001`。
